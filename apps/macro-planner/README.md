# PFC・TDEE計算 macro-planner

アカウント不要・登録なしの、**端末内で完結するPFC/TDEE計算＋食事ログアプリ**。
身長体重・活動量・目標を入れるだけで、**1日の目標カロリーとPFC（タンパク質/脂質/炭水化物）**を計算し、食事ログで残量をリアルタイムに追えます。

ライブ: https://hiiirano.github.io/public-apps/macro-planner/

## なぜこれを作ったか

カロリー計算アプリの定番（MyFitnessPal 等）は便利ですが、(1) アカウント登録が前提、(2) 食品DB・広告・有料プラン、(3) 計算過程がブラックボックス、という弱点があります。
macro-planner はその真逆を狙います。

- **アカウント不要**: 入力するだけ。計算は端末内で完結。
- **プライバシーファースト**: データはこの端末のブラウザ（localStorage）だけ。サーバーには一切送りません。
- **式が透明**: Mifflin-St Jeor 式と Atwater 係数をそのまま使い、純関数でテスト済み。誰でも検証できます。
- **軽い**: 静的SPA・無料・インストール不要・オフライン動作。

## 主な機能

- プロフィール入力で **BMR / TDEE / 目標カロリー / 目標PFC / BMI** をリアルタイム計算
- 目標（減量 −20% / 維持 / 増量 +10%）・活動量・タンパク質(g/kg)・脂質割合(%)を調整可能
- 食事ログ：日付ごとに食品を追加／削除、**残量をバー表示**（超過は赤）
- 任意の LLM 連携：食事メモ（例「鶏むね肉200g、白米150g、卵2個」）から各食品の kcal/PFC を推定して追加（OpenAI互換・APIキーは端末内のみ）
- JSON エクスポート／インポート（バックアップ・端末間移行）

## 決定的コア（純関数）

`src/macro.ts` は実行時 LLM 不要のロジック。見せ場は **TDEE → 目標カロリー → PFC配分**：

```
1. BMR = Mifflin-St Jeor（男性 +5 / 女性 −161）
2. TDEE = BMR × 活動係数（1.2〜1.9）
3. 目標カロリー = TDEE × (1 + 目標補正: cut−20%/maintain0/bulk+10%)
4. P = 体重比(g/kg) / F = カロリー比(%) ÷9 / C = 残りカロリー ÷4（0未満にしない）
```

炭水化物は残りカロリーから逆算するので、高タンパク・高脂質でも 0 で止まり負にはなりません。
すべて端末日付・環境に非依存で、ユニットテスト済み：

```bash
npm install
npm test
```

## 開発

```bash
npm run dev      # 開発サーバ
npm run build    # 型チェック + 本番ビルド
```

データはすべてブラウザの localStorage に保存されます（キー: `macro.profile` / `macro.log` / `macro.llm`）。外部サーバーには送信されません。
本アプリは一般的な推定式による目安であり、医療・栄養指導ではありません。
