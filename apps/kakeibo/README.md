# 家計簿 kakeibo

アカウント不要・口座連携なしの、**端末内で完結するマインドフル家計簿**。
支出を伝統的な4分類（必需／娯楽／文化／予備）で記録し、**予算ペース判定**で「このままだと月末に使いすぎる」を先回りで警告します。

ライブ: https://hiiirano.github.io/public-apps/kakeibo/

## なぜこれを作ったか

自動連携型の家計簿は便利ですが、(1) 口座・カード連携が前提でプライバシーが気になる、(2) 自動集計に任せて「お金と向き合わなくなる」、という弱点があります。
kakeibo はその真逆を狙います。

- **プライバシーファースト**: データはこの端末のブラウザ（localStorage）だけに保存。サーバー送信ゼロ。
- **考える家計簿**: 手入力＋4分類＋月末リフレクションで、支出の意味と向き合う。
- **軽い**: 静的SPA・無料・インストール不要・オフライン動作。

## 主な機能

- 月ごとの予算設定（収入 − 貯金目標 ＝ 使えるお金 を4分類へ配分）
- 支出の記録／編集／削除（日付・金額・分類・メモ）
- ダッシュボード：使えるお金・支出・残り・貯金見込み
- **分類別の予算ペース判定**（バーンレート予測で月末着地を先読み）
- JSON / CSV エクスポート（バックアップ・端末間移行）
- 任意の LLM 連携：レシート/明細テキストから支出を自動抽出（OpenAI互換・APIキーは端末内のみ）

## 決定的コア（純関数）

`src/kakeibo.ts` は実行時 LLM 不要のロジック。月末着地予測は経過日数からの線形外挿：

```
経過日数 = clamp(今日の日, 1, 月の日数)
月末予測 = これまでの支出 / 経過日数 × 月の日数
```

ユニットテスト（基準日を引数化し端末日付に非依存）：

```bash
npm install
npm test
```

## 開発

```bash
npm run dev      # 開発サーバ
npm run build    # 型チェック + 本番ビルド
```

データはすべてブラウザの localStorage に保存されます（キー: `kakeibo.expenses` / `kakeibo.budgets` / `kakeibo.llm`）。
