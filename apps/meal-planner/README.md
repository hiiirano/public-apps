# 🍽️ 週間献立プランナー

家族の条件を選ぶだけで、**1週間分の献立**と**買い物リスト**を自動作成する静的Webアプリ。
登録不要・無料・オフラインでも動作します。

> 「献立／レシピ」という日常で需要の高いテーマを対象にした実用アプリ第1号。

## 特徴

- **入力**：人数（大人/子ども）・ジャンル傾向（和/洋/中）・除外アレルギー・NG食材・冷蔵庫の残り・主菜＋副菜/時短の切替
- **出力**：7日分の献立カード＋カテゴリ別の買い物リスト（人数で按分・同じ材料は合算）＋ひとことコメント
- **ハイブリッド生成**：
  - コア＝レシピDB＋ルールで**決定的に生成**（実行時のLLM不要・無料・オフライン可）
  - 拡張＝任意でLLM APIキーを入れると献立にAIアドバイスが付く（キー無しでもコアは全機能動作）
- アレルギー／NG食材は**ハードフィルタ**（必ず除外）。主菜の週内重複・カテゴリ偏りも回避。

## 使い方（ローカル）

```bash
cd apps/meal-planner
npm install
npm run dev       # 開発サーバ
# または
npm run build && npm run preview   # 本番ビルドを確認
npm test          # 生成ロジック・買い物集約のテスト
```

ブラウザで表示されたURL（既定 http://localhost:5173 / preview は 4173）を開く。

## 公開（GitHub Pages 等）

`vite.config.ts` の `base: "./"` で相対パス出力済み。`npm run build` の `dist/` をそのまま任意の静的ホスティングに置けます。

## AI拡張の設定（任意）

画面下部「🤖 AI拡張（任意）」を開き、OpenAI互換エンドポイント・モデル・APIキーを入力して保存。
**キーはこのブラウザの localStorage のみに保存**され、入力した提供元へ直接送信されます（自前サーバは経由しません）。

## 構成

| ファイル | 役割 |
|---|---|
| `src/types.ts` | 型定義 |
| `src/data/recipes.ts` | レシピDB（主菜24／副菜16） |
| `src/generator.ts` | 週間献立の生成ロジック（決定的・ハードフィルタ） |
| `src/shopping.ts` | 買い物リスト集約（按分・合算・カテゴリ整列） |
| `src/plan.ts` | 生成＋集約の合成 |
| `src/llm.ts` | 任意のLLM拡張（OpenAI互換） |
| `src/main.ts` / `src/style.css` | UI |

詳細な仕様は [`SPEC.md`](./SPEC.md) を参照。
