# 週間献立プランナー — MVP仕様（SPEC）

> 実用アプリ #1 / 段取りは beads で管理。
> このSPECは下流タスク（スキャフォールド/データ/生成ロジック/買い物リスト/UI）の前提。

## ゴール
家族の入力から **7日分の献立＋集約した買い物リスト** を生成する小さなWebアプリ。
「多くの人に役立つ」を最優先 → **無料・APIキー不要・オフラインでも動く** を死守する。
（AIは「データ作り」と「開発」に使う。アプリ本体は実行時にLLMを必須にしない）

## 生成エンジン：ハイブリッド（確定）
- **コア（MVP完了ゲート）**: レシピDB＋ルールベースで **決定的に** 週間献立を生成。実行時LLM不要。
- **拡張（任意）**: ユーザーがLLM APIキーを入れると、献立バリエーション・栄養コメントが賢くなる。
  - キーが無くても **コアはフル動作**（graceful degradation）。拡張はあくまで上乗せ。
  - キーはブラウザの localStorage のみ保持。送信先は選んだprovider直（自前バックエンドに通さない）。

## 入力
- 人数（大人/子ども）
- アレルギー / NG食材（チェック＋自由入力）… **絶対除外（ハードフィルタ）**
- ジャンル傾向（和/洋/中/時短 など）
- 1日の品数（主菜のみ / 主菜＋副菜）
- 冷蔵庫の残り（任意）… コア＝可能なら優先採用 / 拡張＝より賢く活用

## 出力
- 7日 ×（主菜［＋副菜］）の献立表
- **カテゴリ別買い物リスト**（材料を名寄せ・数量合算）
- 簡易な栄養/バランスコメント

## 生成ルール（コア）
- アレルギー/NG食材を含むレシピは候補から除外（ハードフィルタ）
- 同一主菜の週内重複を回避
- 主菜カテゴリ（肉/魚/麺/...）の偏りを回避

## フォームファクタ / 技術
- **静的クライアントSPA（バックエンド無し）**。GitHub Pages で即公開可能。スマホ対応。
- Vite + TypeScript + 軽量CSS。状態は localStorage。
- LLM拡張は provider-agnostic な fetch 層に隔離。

## データ
- 最小シードから開始（目安：主菜〜40 / 副菜〜30）。
- 各レシピ：材料（名称/数量/カテゴリ）・タグ（ジャンル/アレルゲン/調理時間）。

## 非ゴール（MVP）
- ユーザーアカウント / サーバ保存 / 栄養の厳密計算 / 課金
