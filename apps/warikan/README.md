# 割り勘 warikan

アカウント不要・登録なしの、**端末内で完結する割り勘/立替精算アプリ**。
旅行や飲み会の立替を入力すると、**誰が誰にいくら払えば精算できるか**を最小の送金回数で計算し、結果を **URL で共有**できます。受け取った相手も登録不要。

ライブ: https://hiiirano.github.io/public-apps/warikan/

## なぜこれを作ったか

割り勘アプリの定番（Splitwise 等）は便利ですが、(1) 参加者全員のアカウント登録が前提、(2) 有料プラン・広告・データがサーバーに乗る、という弱点があります。
warikan はその真逆を狙います。

- **アカウント不要**: 幹事1人が入力し、結果を URL で共有するだけ。受け取り側も登録なしで見られる。
- **プライバシーファースト**: データはこの端末のブラウザ（localStorage）だけに保存。共有は状態をURLにエンコードして渡すだけで、誰のサーバーにも保存されない。
- **軽い**: 静的SPA・無料・インストール不要・オフライン動作。

## 主な機能

- グループ作成・メンバー管理（旅行/飲み会など複数グループ）
- 立替支出の記録／編集／削除（項目・金額・立替者・対象メンバー）
- 精算結果：各人の収支（立替／負担／差引）と、**最小送金リスト**（誰 → 誰 いくら）
- **URL 共有**：リンクを送ると相手はアカウント不要で結果を確認・取り込み
- JSON エクスポート／インポート（バックアップ・端末間移行）
- 任意の LLM 連携：明細テキストから支出を自動抽出（OpenAI互換・APIキーは端末内のみ）

## 決定的コア（純関数）

`src/warikan.ts` は実行時 LLM 不要のロジック。見せ場は **債務最小化（貪欲法）**：

```
1. 各メンバーの収支 net = 立替合計 − 負担合計
2. 債務者(net<0)と債権者(net>0)を額の大きい順に貪欲マッチング
3. 最大の債務者→最大の債権者へ min(債務,債権) を送金、を全員ゼロまで繰り返す
```

「全員が全員に払う」素朴割り勘の O(N²) 送金を大幅に削減します。均等割りの端数は id 昇順で1円ずつ配分して決定的（合計は必ず元の金額に一致）。

ユニットテスト（端末日付・端数に非依存）：

```bash
npm install
npm test
```

## 開発

```bash
npm run dev      # 開発サーバ
npm run build    # 型チェック + 本番ビルド
```

データはすべてブラウザの localStorage に保存されます（キー: `warikan.groups` / `warikan.current` / `warikan.llm`）。共有 URL は状態を `#data=` ハッシュにエンコードしたもので、外部サーバーには送信されません。
